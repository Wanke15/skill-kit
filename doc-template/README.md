# Doc Template Skill 使用示例

## 示例输入

```
请使用doc-template生成一个文档：

标题：API设计最佳实践

章节：
1. 引言 - 为什么API设计很重要
2. 核心原则 - RESTful、版本控制、错误处理
3. 安全考虑 - 认证、授权、数据保护
4. 实施建议 - 从原型到生产的步骤
```

## 示例输出

系统会生成一个完整的HTML文档，包含：

1. **Header区域**：主标题和副标题
2. **导航栏**：粘性定位，点击跳转到对应章节
3. **目录**：双列布局的目录
4. **内容章节**：使用合适的组件展示

## 组件使用场景

| 组件 | 适用场景 |
|------|----------|
| `highlight-box` | 核心观点、重要提示 |
| `info-box` | 补充说明、注意事项 |
| `cards-grid` | 并列概念、优缺点对比 |
| `definition-list` | 术语定义、概念解释 |
| `step-list` | 操作步骤、实施流程 |
| `timeline` | 发展历程、阶段说明 |
| `comparison-box` | 二选一对比、方案比较 |
| `method-badge` | 方法标签、技术分类 |
| `figure-container` | 图片展示、图表说明 |

## 自定义颜色

模板使用CSS变量，可在生成时调整：

```css
--primary-color: #d97706;    /* 主色调 - 橙色 */
--secondary-color: #1a1a2e;  /* 次要色 - 深蓝 */
--accent-color: #f59e0b;     /* 强调色 - 金色 */
```

如需更改配色，在生成HTML时修改这些变量即可。
